# go_echarts_tools - 甘特图与数据可视化工具

一个基于 Go + Gin + ECharts 的轻量级数据可视化平台，支持 CSV/XLSX 文件自动解析、智能列映射、甘特图和多种数据图表生成。

## 🚀 快速开始

### 安装和运行

```bash
# 1. 进入项目目录
cd go_echarts_tools

# 2. 下载依赖
go mod download

# 3. 运行应用
go run main.go

# 4. 打开浏览器
# 访问: http://localhost:8080
```

### 编译生产版本

```bash
# 编译所有平台（macOS、Linux、Windows）
bash build.sh

# 编译完成后在 dist/ 目录中找到对应平台的可执行文件
./dist/gantt-darwin-arm64/gantt
```

## 📊 使用指南

### 方案 1: 甘特图（项目进度追踪）

**适用场景**: 项目管理、任务规划、里程碑追踪

**使用步骤**：

1. **访问甘特图页面**
   ```
   http://localhost:8080/gantt
   ```

2. **上传数据文件** （支持 CSV 和 XLSX）
   - 点击"选择文件"上传 CSV 或 XLSX 文件
   - 或点击"加载演示数据"查看效果示例

3. **配置列映射**
   - 系统自动推荐列映射（智能识别）
   - 调整必需字段：
     - 📌 **任务名称** (TaskName)
     - 📅 **开始日期** (StartDate)
     - 📅 **结束日期** (EndDate)
   - 可选字段：项目、描述、里程碑、责任人等

4. **生成甘特图**
   - 点击"生成图表"按钮
   - 查看交互式甘特图展示
   - 支持按项目分组、按时间顺序排列

#### 数据格式示例（CSV）

```csv
Project,Task,StartDate,EndDate,Description,Milestone,MilestoneDate,Owner
Website,需求分析,2026-04-01,2026-04-05,定义项目范围,范围确认,2026-04-05,Alice
Website,设计,2026-04-06,2026-04-12,UI/UX 设计,设计评审,2026-04-11,Bob
Website,前端开发,2026-04-13,2026-04-24,页面开发,基础功能,2026-04-24,Charlie
Website,后端开发,2026-04-14,2026-04-26,API 开发,API 完成,2026-04-26,Diana
Website,集成测试,2026-04-27,2026-05-02,集成和测试,上线准备,2026-05-02,Eric
```

#### 高级功能

- 📊 **计划 vs 实际对比**: 包括 PlanStartDate 和 PlanEndDate 列
- 📈 **统计信息**: 自动计算平均时长、总工期、最大任务时长
- 🎨 **颜色分组**: 按 ColorGroup 列自动着色
- 📋 **任务编号**: 可选显示任务序号

### 方案 2: 多图表可视化（通用数据分析）

**适用场景**: 业务数据分析、销售数据、财务报告、多维数据展示

**使用步骤**：

1. **访问可视化页面**
   ```
   http://localhost:8080/viz
   ```

2. **上传数据文件**
   - 支持 CSV/XLSX，自动识别数据结构
   - 或加载演示数据

3. **选择图表类型**
   - 📊 **柱状图** / **条形图**: 分类对比
   - 📈 **折线图**: 趋势分析
   - 🍪 **饼图** / **环形图**: 占比展示
   - 🔲 **散点图**: 相关性分析
   - 📊 **仓库图**: 部分-整体关系
   - 🌳 **树形图**: 层级结构
   - 🌊 **桑基图**: 流向关系

   - **X 轴** / **分类轴**: 报告维度
   - **Y 轴** / **数值轴**: 数值指标
   - **分组**: 按其他维度分组显示
   - **聚合**: 求和、平均值、最大值等

5. **应用样式**
   - 选择主题（亮色 / 暗色）
   - 调整图表显示选项
   - 自定义标题和描述

#### 数据格式示例（CSV）

Cloud,2026-03,145,84,61

## 📋 支持的数据格式

### CSV 文件
- **编码**: UTF-8（推荐）
- **分隔符**: 逗号 (,)
- **首行**: 列名必填
- **日期格式**: 支持多种格式
  - ISO 8601: `2026-04-01`, `2026/04/01`
  - 点分格式: `2026.4.1`
  - 斜杠格式: `4/1/2026`, `04-01-2026`
  - 等等（超过 20 种格式）

### Excel 文件
- **格式**: XLSX（推荐）, XLSM, XLS
- **工作表**: 自动读取第一个工作表
- **要求**: 同 CSV 格式要求

## 📞 常见问题

### Q: 上传文件后看不到数据映射表单？
**A**: 检查文件格式是否正确（仅支持 CSV/XLSX），且至少包含表头行和一行数据。

### Q: 日期解析出现空值？
**A**: 确保日期格式符合支持的格式列表，推荐使用 ISO 8601 格式 (YYYY-MM-DD)。

### Q: 如何导出图表？
**A**: 当前版本支持截图（浏览器内置）或 HTML 导出。计划支持 PDF/PNG 导出。

### Q: 数据会被保存到服务器吗？
**A**: 默认内存存储，重启后丢失。如需持久化，请集成数据库（见扩展方向）。

# 文件结构
main.go                          ← 纯入口，仅 embed + 启动 server
internal/
  model/types.go                 ← 所有共享数据类型 (Task, Stats, Dataset, MappingConfig…)
  data/
    parse.go                     ← CSV/XLSX 解析、日期解析、Cell 工具函数
    store.go                     ← 数据集内存存储（线程安全）
  charts/
    chart.go                     ← ChartBuilder 接口 + 注册表 (Register/Get/All)
    gantt/
      gantt.go                   ← 甘特图实现，init() 自动注册
  server/
    server.go                    ← gin 路由、静态文件、模板装配
    handlers.go                  ← HTTP 处理器（home/demo/upload/chart）
    render.go                    ← renderWorkspace / renderMapper 渲染辅助


# 后续扩展
## 后续扩展新图表类型只需：

在 internal/charts/<新类型>/ 下实现 ChartBuilder 接口
在该包的 init() 中调用 charts.Register(New())
在 server.go 中加一行 _ "gantt/internal/charts/<新类型>" 侧效导入
